# 🚀 중고거래 경매 플랫폼 (Auction Platform) 리팩토링 및 기능 완성 계획서

## 📋 1. 프로젝트 개요

본 문서는 기존 중고거래 경매 플랫폼의 UI/UX를 전면 리팩토링하고, 핵심 미구현 기능을 완성하여 서비스의 완성도와 사용자 경험을 극대화하는 것을 목표로 합니다.

- **현재 화면 목록:** (상품 목록, 상품 상세, 상품 등록/수정 등) 기존에 구현된 화면의 **HTML/JSX 구조 및 CSS 스타일**을 분석하여 디자인 문제점을 파악하고 재설계합니다.
- **컴포넌트 구조:** 현재 사용 중인 컴포넌트들의 재사용 가능성 여부를 판단하고, 재구성합니다.
- **디자인 개선:** 중고거래 경매 플랫폼에 어울리는 디자인을 선택합니다. 현재 @mui/material 7.3.6 사용중 라이브러리 추가 고려가능
- **기존 로직:** 상품 상태 변경 로직, 입찰 로직 등 **백엔드와 통신하는 기존 API 호출 로직**은 최대한 유지하면서 프론트엔드 UI/UX만 개선합니다.
- **미구현 화면:** 찜목록,주문/낙찰 내역 등 사용자 핵심 흐름 화면 디자인 및 구현.

## ⚙️ 2. 핵심 기능별 구현 및 리팩토링 상세 계획

### 2.1. 상품 목록 및 상세 보기 화면 개선(Product Listing & Detail)

| 기능 ID      | 기능명                  | 구현 내용                                                                                                                                                             |
| :----------- | :---------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **FEAT-001** | 상품 목록 보기 (List)   | 1. 경매 상태(진행 중, 마감, 유찰, 취소)에 따른 **필터링 및 정렬** 기능 UI 개선. <br> 2. 상품 카드에 **최고 입찰가**와 **마감 임박 타이머** 표시.                      |
| **FEAT-002** | 상품 상세 보기 (Detail) | 1. 상품 정보, 현재 경매 정보, 이전 경매 이력을 **탭 또는 섹션**으로 분리하여 정보 접근성 확보. <br> 2. 현재 경매가 있다면 **'경매 참여하기'** 버튼을 시각적으로 강조. |

### 2.2. 경매 상세 및 입찰 기능 화면 개선(Auction Bidding)

| 기능 ID      | 기능명         | 구현 내용                                                                                                                                            |
| :----------- | :------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------- |
| **FEAT-003** | 경매 상세 보기 | 1. **Socket.IO 연동**을 통한 **실시간 입찰가 및 타이머** 업데이트 로직 구현. <br> 2. 입찰 기록(시간, 금액) 목록을 실시간으로 보여주는 컴포넌트 개발. |
| **FEAT-004** | 입찰 진행      | 1. 최소 입찰 단위 안내 및 예치금 잔액을 보여주는 UI 개선. <br> 2. 유효성 검사 (최소 입찰 금액, 예치금 부족 등) 강화.                                 |

### 2.3. 상품 등록 및 경매 관리 (Registration & Management)

| 기능 ID      | 기능명      | 구현 내용                                                                                                                                                                                    |
| :----------- | :---------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **FEAT-005** | 상품 등록   | 1. 상품 정보 (제목, 사진, 설명)와 경매 정보 (시작가, 시작일, 마감일)를 분리하여 **단계별 입력 폼**으로 구현. <br> 2. **드래그 앤 드롭**을 지원하는 다중 이미지 업로드 컴포넌트 도입(후순위). |
| **FEAT-006** | 상품 수정   | 경매 **진행 상태**에 따라 수정 가능한 필드를 제한하는 로직 구현.                                                                                                                             |
| **FEAT-007** | 경매 재등록 | 유찰 또는 취소된 상품의 경우, 기존 상품 정보를 불러와 **경매 정보만 새로 설정**하여 등록하는 기능 구현 (재경매).                                                                             |

### 2.4. 결제 및 예치금 관리 (Payment & Deposit)

| 기능 ID      | 기능명           | 구현 내용                                                                                                                              |
| :----------- | :--------------- | :------------------------------------------------------------------------------------------------------------------------------------- |
| **FEAT-008** | 예치금 충전      | 1. **토스페이 위젯(Toss Payments SDK/API)** 연동을 통한 안전하고 편리한 충전 기능 구현. <br> 2. 충전 금액 선택 UI 및 결제 로직 최적화. |
| **FEAT-009** | 구매 대기 주문   | 1. 내가 낙찰받은 상품 목록 확인 (구매 대기 상태). <br> 2. 낙찰된 주문에 대한 **최종 결제** 기능 구현 (예치금 차감 로직 필수).          |
| **FEAT-010** | 예치금 내역 보기 | 입출금 내역을 명확하게 보여주는 테이블 및 검색/필터링 기능 구현.                                                                       |

### 2.5. 찜 목록 및 마이페이지 (Favorites & MyPage)

| 기능 ID      | 기능명              | 구현 내용                                                                                                              |
| :----------- | :------------------ | :--------------------------------------------------------------------------------------------------------------------- |
| **FEAT-011** | 찜 목록 보기        | 찜한 상품 목록 표시 및 해당 상품의 **현재 경매 상태, 최고 입찰가**를 실시간에 가깝게 표시하여 사용자 추적 용이성 향상. |
| **FEAT-012** | 마이페이지 대시보드 | 예치금 잔액, 입찰 진행 중인 목록, 낙찰/구매 내역을 통합하여 볼 수 있는 대시보드 구현.                                  |
